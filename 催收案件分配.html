🎲 随机分配
每个案件生成随机数 00-99，按区间分配机构
背景：假设S1每天需要分配1000个案件，需要分配30天，每天分给ABCD四家机构。机构份额分别为A-40%, B-30%, C-20%, D-10%。采样使用6月CL S1真实新增案件数据，每日采样1000件（当天TOP100最大金额 + TOP100最小金额 + 800件中部随机采样）。

算法逻辑

每天分配1000个案件 → 给每个案件生成一个随机数(00-99)：
00-39 → A机构(40%) | 40-69 → B机构(30%) | 70-89 → C机构(20%) | 90-99 → D机构(10%)

🐍 蛇形分配
案件金额降序 → 蛇形轮转 → A4→B3→C2→D1→D1→C2→B3→A4
背景：假设S1每天需要分配1000个案件，需要分配30天，每天分给ABCD四家机构。机构份额分别为A-40%, B-30%, C-20%, D-10%。采样使用6月CL S1真实新增案件数据，每日采样1000件（当天TOP100最大金额 + TOP100最小金额 + 800件中部随机采样）。

算法逻辑

案件金额从大到小排序，从最大的金额开始按照份额的比例，蛇形分案个数：
A分4个 → B分3个 → C分2个 → D分1个 → 折返 → D分1个 → C分2个 → B分3个 → A分4个 → 循环...

🎯 贪心算法
局部最优选择组成全局最优解 · 大案优先 + 动态优先级 + 容量上限103%
背景：假设S1每天需要分配1000个案件，需要分配30天，每天分给ABCD四家机构。机构份额分别为A-40%, B-30%, C-20%, D-10%。采样使用6月CL S1真实新增案件数据，每日采样1000件（当天TOP100最大金额 + TOP100最小金额 + 800件中部随机采样）。

算法逻辑

① 案件金额从大到小排序（大案优先，避免大案后期塞不进去）
② 第1轮（前4案）：按份额比例从大到小的优先级分配（A→B→C→D各1件）
③ 第2轮起（第5案开始）：计算 priority = 已分金额 ÷ 机构份额（值越小优先级越高）
④ 每案分配给 priority 最小的机构，判断当月容量上限 ≤ 103%
⑤ 每分配满4案完成一轮，继续下一轮

🏆 算法对比
同一天案件 · 3种算法 × 3种规模 · 对比户数/金额公平度

采样规则

基于当日真实新增分布（t100max / t100min / p50）生成 N 件案件：
10% 极大值（t100max ~ daily_max 区间）+ 10% 极小值（daily_min ~ t100min 区间）+ 80% 中部随机（t100min ~ t100max，以 P50 对半分布）

📐 指标口径

户数公平 = 1−CV(各机构户数占比/目标份额)  |  金额公平 = 1−CV(各机构金额占比/目标份额)
户均均衡 = 1−CV(各机构户均金额)  |  极差均衡 = 1−CV(各机构极差)
四个指标均为越高越好。CV=σ/μ。其中"极差"=每机构最大案件−最小案件，各机构极差的CV越小 → 各机构金额跨度越接近 → 极差均衡越高 → 极端案件未被某家独吞。
🔄 交替分配
M2+ 委外催收场景 · 同一案件不连续两个月分给同一机构
背景：假设S1每天需要分配1000个案件，需要分配30天，每天分给ABCD四家机构。机构份额分别为A-40%, B-30%, C-20%, D-10%。采样使用6月CL S1真实新增案件数据，每日采样1000件。

算法逻辑

① 案件金额从大到小排序（与贪心算法相同）
② 计算 priority = 已分金额 ÷ 机构份额，值越小优先级越高
③ 判断：该案件上个月分配的机构是否是优先级最高的机构？
   → 是：跳过，分配给优先级第二的机构
   → 否：分配给优先级最高的机构
④ 容量上限 103%

💧 注水分案
员工维度负载均衡 · 每天分案后各员工库内案件数量和金额趋同

算法逻辑

① 第2天起，统计前一天员工的在库金额（= 前一天分案 − 前一天回收）
② 当天案件金额从大到小排序（大案优先分配）
③ priority = 在库金额 + 今日已分配金额（值越小优先级越高）
④ 逐件分配给 priority 最小的员工
⑤ 容量上限：当天分配 ≤ (当天总额 ÷ 员工数) × 1.3
   （防止前一天休息员工因在库少而一次性拿到过多案件）
