## 2 基础分配方法

**背景**：每天 1,000 件 × 30 天，ABCD 四家机构（A=40% / B=30% / C=20% / D=10%）。

### 2.1 随机分配

每案生成随机数 00-99：00-39→A | 40-69→B | 70-89→C | 90-99→D。

### 2.2 蛇形分配

案件降序排列 → 蛇形轮转 A4→B3→C2→D1→D1→C2→B3→A4 → 循环。

### 2.3 贪心算法

案件降序排列(大案优先) → 第1轮按份额 A→B→C→D 各 1 件 → 第2轮起 `priority = 已分金额 ÷ 份额`，最小者优先 → 容量上限 103%。

### 2.4 算法对比

| 维度 | 随机 | 蛇形 | 贪心 |
|------|------|------|------|
| 户数公平 | ★★★ | ★★★ | ★★ |
| 金额公平 | ★ | ★★ | ★★★ |
| 大额控制 | ❌ | ⚠️ | ✅ |
| 容量上限 | 无 | 无 | 103% |
| 适用 | 无差异 | 差异中等 | 差异极大(S1) |

### 2.5 公平性指标

| 指标 | 公式 |
|------|------|
| 户数公平 | 1 − CV(各机构户数占比 ÷ 目标份额) |
| 金额公平 | 1 − CV(各机构金额占比 ÷ 目标份额) |
| 户均均衡 | 1 − CV(各机构户均金额) |
| 极差均衡 | 1 − CV(各机构极差)，极差 = max−min |

> 四个指标均为越高越好，CV = σ/μ。

## 3 催收效率提升

### 3.1 交替分配

案件降序 → `priority = 已分金额 ÷ 份额` → 判断：该案件上月机构是否是优先级最高的机构？是则跳过，分配给优先级第二的机构；否则正常分配 → 容量上限 103%。用于 M2+ 委外催收强制换手。

### 3.2 注水分案

每天 100 件 × 10 名员工。Day1 每人固定 10 件 × 金额均衡。Day1 回收后（回收率差异 2%~8%），Day2 初始在库出现差异。

Day2 起：案件降序 → `priority = 在库金额 + 今日已分金额` → 逐件分配给 priority 最小的员工 → 容量 ≤ 人均×1.3（防休息偏差）。注水后分案后在库再次拉平，Day5 各员工在库趋近。
